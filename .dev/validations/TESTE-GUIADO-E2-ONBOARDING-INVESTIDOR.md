# Teste Guiado — E2: Onboarding Investidor em 2 Etapas

**Data:** 06/04/2026
**Status do Épico:** Concluído
**Objetivo:** Validar que o fluxo de entrada do investidor é simples, rápido e funcional.

---

## O que é este épico?

O Épico 2 criou o **fluxo de onboarding do investidor** em duas etapas: primeiro o investidor cria sua **Tese de Investimento** (o que procura), depois acessa o **Radar** que mostra empresas compatíveis com seus critérios. A ideia é reduzir o tempo entre cadastro e valor — o investidor rapidamente enxerga oportunidades relevantes.

---

## O que você deve conseguir fazer

- Criar uma tese de investimento com seus critérios (setores, regiões, faixa de investimento, etc.).
- Editar e excluir teses existentes.
- Visualizar o Radar com empresas que fazem match com sua tese.
- Entender rapidamente quais oportunidades são mais relevantes.

---

## Checklist de Validação (10 tarefas)

Marque cada item após testar. Se encontrar problema, descreva brevemente ao lado.

| # | Tarefa | OK? | Observação |
|---|--------|-----|------------|
| 1 | Acesse a seção **Teses** no menu lateral. Verifique se a página carrega corretamente. | ☐ | |
| 2 | Clique em **Criar nova tese**. Verifique se o formulário guiado (wizard multi-step) aparece. | ☐ | |
| 3 | Preencha o **Passo 1** da tese (setores de interesse, regiões, faixa de cheque/ticket). Avance para o próximo passo. | ☐ | |
| 4 | Preencha o **Passo 2** da tese (critérios adicionais). Finalize a criação e confirme que a tese aparece na listagem. | ☐ | |
| 5 | Abra a tese criada e clique em **Editar**. Altere um campo (ex: região) e salve. Confirme que a alteração foi persistida. | ☐ | |
| 6 | Acesse o **Radar** no menu lateral. Verifique se a página carrega e exibe resultados de empresas compatíveis com sua tese. | ☐ | |
| 7 | No Radar, verifique se cada empresa exibe informações resumidas claras (setor, região, score de compatibilidade ou similar). | ☐ | |
| 8 | Verifique se existe um **CTA (botão de ação)** visível em cada card do Radar (ex: "Ver detalhes", "Demonstrar interesse"). | ☐ | |
| 9 | Volte à listagem de teses e tente **excluir** uma tese. Confirme que o sistema pede confirmação antes de deletar. | ☐ | |
| 10 | Verifique se o formulário de tese **não exibe JSON técnico** nem campos confusos — tudo deve ser legível para o usuário final. | ☐ | |
