## Resumo

Transformar a Mary AI de drawer/overlay (`Sheet`) para sidebar push que empurra o conteúdo para a esquerda, com context provider, CTAs por perfil, saudação e disclaimer conforme Excalidraw.

**4 fases de implementação**, cada uma atômica e commitável isoladamente.

**Risco:** Baixo
**Justificativa de ordem:** O provider é a fundação — todas as fases seguintes dependem dele para estado (`isOpen`, `toggle`, `currentPage`, `currentProfile`). Criar primeiro permite validar a API de contexto antes de mexer no layout.

|------|---------|
| CRIAR | `src/components/providers/MaryAiProvider.tsx` |
| MODIFICAR | `src/components/providers/index.ts` (re-export) |

### Escopo

1. Criar `MaryAiProvider` com estado:
   - `isOpen: boolean` (default `false`)
   - `toggle(): void`
   - `currentPage: string` (derivado de `usePathname()`)
   - `currentProfile: 'investor' | 'asset' | 'advisor' | null` (derivado de `OrganizationProvider`)

## Fase 2 — MaryAiSidebar (novo componente)

**Risco:** Baixo
**Justificativa de ordem:** Criar o componente de sidebar antes de integrar no layout permite testá-lo isoladamente. Não altera nenhum componente existente ainda.

| Ação | Arquivo |

| MODIFICAR | `src/components/mary-ai/MaryAiQuickChatSheet.tsx` (extrair lógica compartilhada) |

### Escopo

1. **Extrair lógica de chat** do `MaryAiQuickChatSheet.tsx` para `MaryAiChatContent.tsx`:

   - Isso permite reutilizar no Sheet (Onboarding) e na Sidebar

   - Remove lógica de chat (agora em `MaryAiChatContent`)
   - Fica como wrapper: `Sheet` + `MaryAiChatContent`

2. **`ProtectedLayoutClient.tsx`:** Refatorar layout de `flex` 2 colunas para 3 zonas:

       "flex-1 flex flex-col min-w-0 overflow-hidden transition-smooth",

## Decisões técnicas consolidadas

| Decisão | Justificativa |
|---------|--------------|
| Botão no header (não floating) | Todos os wireframes Excalidraw mostram `Mary AI ▼` no header |
| `w-96` (384px) | Padrão de sidebars de chat, compatível com layout 3 colunas |
| Estado `isOpen` persiste entre rotas | Sidebar contextual precisa estar aberta ao navegar |
| CTAs por perfil (não por tela) no MVP | Excalidraw define CTAs apenas por perfil; contextualidade por tela via `currentPage` ao LLM |
| Chat existente migra sem novo backend | H0.5 é refatoração UI; backend de persistência é spec separada |
| Breakpoints: 1280px / 768px | 3 camadas responsivas: tudo coexiste / adaptado / mobile fullscreen |
| Componente de chat compartilhado | `MaryAiChatContent` reutilizado em Sidebar e Sheet (Onboarding) |
| `transition-smooth` (300ms) | Token existente em `globals.css`, conforme CLAUDE.md |
