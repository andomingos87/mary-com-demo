**Mary Taxonomy v1**
Estrutura inicial robusta e legalmente livre (sem restrição de copyright), mas compatível com os padrões globais (NAICS) e nacionais (CNAE).
Abaixo está a **versão base hierárquica** — 12 macrosetores, cada um com subsetores principais, e equivalências aproximadas NAICS + CNAE.
> ⚙️  Esta versão serve como **Layer 2 (comparabilidade e matching)** na Mary, interoperando com CNAE (Layer 1 compliance) e servindo de base para IA e Readiness Score®.

## 🗂️  Estrutura de tabela sugerida (Supabase)

```sql
CREATE TABLE mary_taxonomy (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  level INT, -- 1=macro, 2=setor, 3=subsetor
  code TEXT UNIQUE,
  name TEXT,
  parent_code TEXT,
  description TEXT,
  naics_codes TEXT[],
  cnae_codes TEXT[],
  created_at TIMESTAMP DEFAULT now()
);
```

---

## 🔗 Uso interno da plataforma

* **Matching AI:** as embeddings e filtros operam sobre o `mary_taxonomy.code`.
* **Interoperabilidade:** sempre que o usuário informar CNPJ, o campo `cnae_principal` será mapeado para um ou mais `mary_code` (ex.: via tabela `mary_taxonomy_mapping`).
* **Relatórios:** dashboards e análises podem usar o `mary_code` como eixo primário, mantendo rastreabilidade legal com `cnae_codes`.
* **Flexibilidade:** como é *open schema*, você pode evoluir (ex.: adicionar “ClimateTech”, “Health Data Infra”, etc.) sem quebrar nada.
