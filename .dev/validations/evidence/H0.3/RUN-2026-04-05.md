### 2.2 Enum `project_status` após migração

Query:

```sql
SELECT e.enumlabel AS value
FROM pg_enum e
JOIN pg_type t ON e.enumtypid=t.oid
WHERE t.typname='project_status'
ORDER BY e.enumsortorder;
```

Resultado inclui os valores H0.3:
- `screening`, `teaser`, `nda`, `cim_dfs`, `ioi`, `management_meetings`, `nbo`, `dd_spa`, `signing`, `cps`, `closing`, `disclosure`, `closed_won`, `closed_lost`

Resultado:
- enum canônico com 14 valores operacionais H0.3 (sem `spa` no tipo ativo)

### 2.3 Contrato de escrita no banco

Query:

```sql
SELECT conname, pg_get_constraintdef(c.oid) AS definition
FROM pg_constraint c
JOIN pg_class r ON r.oid=c.conrelid
JOIN pg_namespace n ON n.oid=r.relnamespace
WHERE r.relname='projects'
  AND n.nspname='public'
  AND conname LIKE 'projects_status%';
```

Resultado:
- constraint ativa: `projects_status_h03_check`
- lista permitida da constraint: os 14 status do contrato H0.3

### 2.4 Persistência e auditoria após DnD

Query de persistência:

```sql
SELECT codename, status::text
FROM public.projects
WHERE codename IN ('h03-scr-test','h03-nda-test')
ORDER BY codename;
```

Resultado:
- `h03-scr-test` => `teaser` (movido de `screening`)
- `h03-nda-test` => `nda`

Query de auditoria:

```sql
SELECT action, metadata->'status_change' AS status_change, created_at
FROM public.audit_logs
WHERE action IN ('project.stage_changed','project.stage_rollback')
ORDER BY created_at DESC
LIMIT 5;
```

Resultado:
- `project.stage_changed` com `from: screening`, `to: teaser`

### 2.5 Fechamento técnico dos itens 2.4/2.5

Query de auditoria de migração:

```sql
SELECT action, metadata->>'event' AS event, metadata->'legacy_mapping' AS legacy_mapping
FROM public.audit_logs
WHERE metadata->>'event' IN ('pipeline.migration_5_to_12', 'pipeline.migration_5_to_12_cleanup')
ORDER BY created_at DESC

```

Resultado:

Resultado:
