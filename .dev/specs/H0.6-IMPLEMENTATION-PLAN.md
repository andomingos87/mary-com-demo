## Decisões tomadas

| Decisão | Resolução | Fonte |
|---------|-----------|-------|
| Persistência Supabase | `onSave` callback é responsabilidade de cada tela; hook orquestra debounce + status + retry | `01_GLOBAL_RULES.md` |
| Debounce por campo vs formulário | Por campo, timer independente de 2000ms | Interface `registerField` + Excalidraw |
| Botões "Próximo" permanecem | São navegação, não persistência. Excalidraw mostra CTAs de avanço em todos os passos | `02_ATIVO.md`, `03_INVESTIDOR.md` |
| Textos de tooltip | Literais do Excalidraw (02-05), catálogo em `tooltips.ts` | `02_ATIVO.md`, `03_INVESTIDOR.md`, `04_ADVISOR.md` |
| MRS "validar" | Padronizar feedback visual para `AutoSaveIndicator`; não criar funcionalidade nova | Spec H0.6 |
| Advisor fora do escopo | Telas não existem. Textos de tooltip podem entrar no catálogo; aplicação nas telas fica para spec do Advisor | PRD v3.0 (❌ AUSENTE) |
| Tooltips em código vs banco | `tooltips.ts` (pragmático). Mary Messaging System fica para spec futura | PRD v3.0 §3.5 |

## Riscos técnicos

| Risco | Prob. | Impacto | Mitigação |
|-------|-------|---------|-----------|
| Server actions não granulares por campo | Média | Alto | Agrupar campos no debounce window e enviar delta. Ou criar actions field-level |
| `ProfileDetailsForm` polimórfico quebra | Baixa | Médio | Testar 3 profile types. Flag `enableAutoSave` |
| Race conditions no auto-save | Média | Médio | AbortController por campo. Último save vence |
| Tooltip overflow em mobile | Média | Baixo | `max-w-xs` + `overflow-wrap`. Testar responsividade |
| Regressão: perda de drafts localStorage | Baixa | Alto | Migration path: load → migrar → limpar |
